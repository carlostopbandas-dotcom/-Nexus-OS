# Story 1.3.5 — Criar atoms: Button, Input, Badge, Avatar

**Status:** Ready for Review | **Sprint 3 — Frontend System** 🟡 | **Agente:** @dev | **Esforço:** 6h

---

## Story Statement

**Como** desenvolvedor do Nexus OS,
**Quero** ter componentes atômicos reutilizáveis para Button, Input, Badge e Avatar,
**Para que** as 8+ variações inline de botão e 10+ variações inline de badge sejam substituídas por componentes consistentes com design tokens e acessibilidade.

---

## Contexto

O inventário atual:
- Botões: 8+ variações inline sem abstração — qualquer mudança visual requer busca em 200+ linhas
- Badges: 10+ variações de cor inline sem padronização
- Inputs: todos sem `aria-label` (WCAG Level A failure)
- Avatar: "CS" hardcoded em Sidebar (não conectado ao Auth)

Após Story 1.1.1 (Auth), o Avatar pode usar `user.email` do Supabase Auth.

[Source: docs/architecture/frontend-spec.md#31-atomos]
[Source: docs/architecture/ux-specialist-review.md#42-componentes-atomicos]

### Avatar — initials do Auth

```typescript
// components/ui/avatar.tsx
import { useAuth } from '@/components/auth/AuthProvider'

function getInitials(email: string): string {
  const name = email.split('@')[0]
  const parts = name.split(/[._-]/)
  return parts.slice(0, 2).map(p => p[0]?.toUpperCase() ?? '').join('')
}

export function Avatar({ size = 'md' }: { size?: 'sm' | 'md' | 'lg' }) {
```
