## Dev Notes

### Extensão de `AuthProvider` — padrão

```typescript
interface AuthContextValue {
  user: User | null
  session: Session | null
  loading: boolean
  userRole: UserRole | null       // NOVO
  profileLoading: boolean         // NOVO — true enquanto busca profile
}

// Dentro do AuthProvider:
useEffect(() => {
  supabase.auth.getSession().then(async ({ data: { session } }) => {
    setSession(session)
    setUser(session?.user ?? null)
    if (session?.user) {
      const { data } = await userProfilesService.getMyProfile()
      setUserRole(data?.role ?? null)
    }
    setLoading(false)
  })
  // onAuthStateChange também deve atualizar userRole
}, [])
```

### `RoleGuard` — uso

```tsx
// Ocultar seção da Sidebar para não-CEO:
<RoleGuard roles={['ceo']}>
  <SidebarItem icon={Users} label="Usuários" to="/users" />
</RoleGuard>

// Proteger página inteira:
<RoleGuard roles={['ceo']} fallback={<Navigate to="/" />}>
  <Users />
</RoleGuard>
```

### Tela "Aguardando Ativação" (AC-6)

Usuário autenticado mas sem `user_profiles` entry → exibir tela de espera, **não** o app:

```tsx
// Em ProtectedRoute ou AppContent:
if (!profileLoading && user && userRole === null) {
  return <PendingActivationPage />  // inline simples — "Sua conta está aguardando ativação pelo CEO"
}
```

### Página `/users` — criação de usuário

Carlos precisa primeiro criar o usuário no Supabase Dashboard (Authentication > Users > Invite). Após o usuário aceitar o invite e logar pela primeira vez, Carlos edita o papel na página `/users`.

Mostrar esse fluxo como instrução inline na página (não é bug, é limitação do Supabase sem Admin API no frontend).

### Ícones a adicionar

`Users` (Sidebar + página) de `lucide-react` — já pode estar importado na Sidebar, verificar.

### Sem nova migration

Esta story não requer migration SQL — `006_multi_user_rls.sql` já cobre o schema completo.
