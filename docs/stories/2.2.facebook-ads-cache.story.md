### Restrição crítica: Supabase Hobby Plan

Supabase Hobby não suporta `pg_cron`. O sync automático usa **Vercel Cron** (mesmo padrão do Shopify), não pg_cron:

```
Vercel Cron (1h) → /api/ads/facebook-sync → Supabase Edge Function ads-facebook-sync
                                                    ↓
                                          ads_campaigns_cache + ads_sync_log
```

A Edge Function `ads-facebook-sync` é **HTTP-invocable** (não usa cron interno Supabase). O Vercel Cron chama o endpoint da Edge Function com um `CRON_SECRET` para autenticar.

### Escopo: VcChic BM apenas

Moriel e Sezo têm lojas pausadas em Ads — **não integrar**. A story cobre exclusivamente o Business Manager VcChic.

### Configuração necessária (pré-condição para funcionar em produção)

Carlos precisa configurar antes do deploy:
1. Meta Business Manager → System Users → criar System User com permissão `ads_read` + `read_insights`
2. Gerar token de longa duração (System User Token — não expira)
3. Adicionar como secret da Edge Function: `FB_ACCESS_TOKEN_VCCHIC_BM`
4. Adicionar o Ad Account ID do BM VcChic: `FB_AD_ACCOUNT_ID_VCCHIC` (formato: `act_123456789`)
5. Adicionar `CRON_SECRET` nas env vars do Vercel + Edge Function

> **O @dev implementa o código completo.** A integração fica desabilitada em produção até Carlos configurar os secrets. A UI exibe estado "Aguardando configuração" quando `FB_ACCESS_TOKEN_VCCHIC_BM` não está setado.

### Autenticação dual no Vercel route

O Vercel route `/api/ads/facebook-sync.ts` aceita dois chamadores com autenticação distinta:

```typescript
// api/ads/facebook-sync.ts (Vercel)
import { createClient } from '@supabase/supabase-js'

export default async function handler(req, res) {
  // Chamador 1: Vercel Cron (server-to-server) — valida x-vercel-cron
  const isCron = req.headers['x-vercel-cron'] === '1'
    || req.headers['x-cron-secret'] === process.env.CRON_SECRET

  if (!isCron) {
    // Chamador 2: Frontend — valida JWT Supabase do usuário logado
    const authHeader = req.headers.authorization
    if (!authHeader) return res.status(401).json({ error: 'Unauthorized' })
    const supabase = createClient(process.env.SUPABASE_URL!, process.env.SUPABASE_ANON_KEY!)
    const { data: { user } } = await supabase.auth.getUser(authHeader.replace('Bearer ', ''))
    if (!user) return res.status(401).json({ error: 'Unauthorized' })
  }

  // Chama Edge Function com CRON_SECRET (server-side, nunca chega ao frontend)
  const edgeFnUrl = `${process.env.SUPABASE_URL}/functions/v1/ads-facebook-sync`
  await fetch(edgeFnUrl, {
    method: 'POST',
    headers: { 'x-cron-secret': process.env.CRON_SECRET! },
  })
  return res.status(200).json({ ok: true })
}
```

`CRON_SECRET` **nunca usa prefixo `NEXT_PUBLIC_`** — permanece server-side.
```
