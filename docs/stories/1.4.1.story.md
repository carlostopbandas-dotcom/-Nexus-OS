# Story 1.4.1 — Configurar ESLint + Prettier

**Status:** Ready | **Sprint 4 — Quality & DevOps** 🟡 | **Agente:** @devops | **Esforço:** 1h

---

## Story Statement

**Como** desenvolvedor do Nexus OS,
**Quero** ter ESLint e Prettier configurados e funcionando,
**Para que** o código siga padrões consistentes e problemas de qualidade sejam detectados automaticamente.

---

## Contexto

O projeto não tem linter configurado. Isso significa que erros de TypeScript, imports não usados, e inconsistências de código passam despercebidos. O ESLint também é pré-requisito para o GitHub Actions (Story 1.4.5).

[Source: docs/architecture/technical-debt-assessment.md — Qualidade de Código 4/10]

---

## Acceptance Criteria

- [ ] **AC-1:** ESLint configurado com regras para TypeScript e React
- [ ] **AC-2:** Prettier configurado com regras consistentes com o estilo atual do projeto
- [ ] **AC-3:** `npm run lint` executa sem erro (ou com apenas warnings conhecidos)
- [ ] **AC-4:** `npm run format` formata o código automaticamente
- [ ] **AC-5:** `.eslintignore` e `.prettierignore` configurados para excluir `node_modules`, `.aiox/`, etc.
- [ ] **AC-6:** Scripts adicionados ao `package.json`

---

## Tasks / Subtasks

- [ ] **Task 1 — Instalar dependências** (AC: 1, 2)
  - `npm install -D eslint @typescript-eslint/eslint-plugin @typescript-eslint/parser eslint-plugin-react eslint-plugin-react-hooks`
  - `npm install -D prettier eslint-config-prettier eslint-plugin-prettier`

- [ ] **Task 2 — Criar `.eslintrc.json`** (AC: 1)
  - Extends: `@typescript-eslint/recommended`, `plugin:react/recommended`, `plugin:react-hooks/recommended`
  - Regras customizadas: `@typescript-eslint/no-explicit-any: warn`

- [ ] **Task 3 — Criar `.prettierrc`** (AC: 2)
  - `{ "semi": false, "singleQuote": true, "tabWidth": 2, "printWidth": 100 }`
  - Ajustar conforme estilo atual do código

- [ ] **Task 4 — Adicionar scripts ao package.json** (AC: 6)
  - `"lint": "eslint . --ext .ts,.tsx"`
  - `"format": "prettier --write 'src/**/*.{ts,tsx}'"`
  - `"lint:fix": "eslint . --ext .ts,.tsx --fix"`

- [ ] **Task 5 — Criar .eslintignore / .prettierignore** (AC: 5)
  - Excluir: `node_modules`, `.aiox/`, `.aiox-core/`, `dist/`, `docs/`

- [ ] **Task 6 — Rodar e corrigir erros críticos** (AC: 3)
  - `npm run lint` — analisar output
  - Corrigir erros críticos (não apenas warnings)
  - Para erros de `any` existentes: usar `// eslint-disable-next-line` temporariamente (Story 1.4.2 limpa)

- [ ] **Task 7 — Commit**
  - `git commit -m "feat: configurar ESLint + Prettier [Story 1.4.1]"`

## Dev Notes

### `.eslintrc.json` base

```json
{
  "extends": [
    "eslint:recommended",
    "plugin:@typescript-eslint/recommended",
    "plugin:react/recommended",
    "plugin:react-hooks/recommended",
    "prettier"
  ],
  "parser": "@typescript-eslint/parser",
  "plugins": ["@typescript-eslint"],
  "rules": {
    "react/react-in-jsx-scope": "off",
    "@typescript-eslint/no-explicit-any": "warn",
    "@typescript-eslint/no-unused-vars": ["warn", { "argsIgnorePattern": "^_" }]
  },
  "settings": {
    "react": { "version": "detect" }
  }
}
```

### Erros esperados no primeiro lint

O projeto tem:
- Múltiplos `any` em tipos (Story 1.4.2 resolve)
- `StoreMetric` duplicada (Story 1.4.3 resolve)
- Alguns imports não usados

Estratégia: corrigir erros `error` (não `warn`) nesta story. Os `warn` de `any` serão resolvidos em 1.4.2.
