# Story 1.2.1 — Instalar React Router v7 e migrar navegação

**Status:** Ready | **Sprint 2 — Architecture & DB Foundation** 🟠 | **Agente:** @dev | **Esforço:** 4h

---

## Story Statement

**Como** Carlos (CEO / único usuário do Nexus OS),
**Quero** que o app use URLs reais para cada página do dashboard,
**Para que** eu possa usar o botão "voltar" do browser, compartilhar links de páginas específicas, e ter deep links funcionais.

---

## Contexto

Hoje o Nexus OS usa `useState('activeTab')` no App.tsx como sistema de navegação. Isso significa:
- URL nunca muda — sempre `/`
- Botão "voltar" do browser não funciona
- Não é possível fazer deep link para uma página específica

Débito remediado: **TD-H3** (Sem React Router), **UX-CRIT-2** (sem deep link, sem back button).

[Source: docs/architecture/technical-debt-assessment.md#td-h3]
[Source: docs/architecture/frontend-spec.md#51-navegacao]
[Source: docs/architecture/ux-specialist-review.md#adrs]

## Dev Notes

### Arquitetura atual a migrar

```typescript
// App.tsx — estado atual
const [activeTab, setActiveTab] = useState<string>('dashboard')

// Sidebar.tsx — padrão atual (simplificado)
<button onClick={() => setActiveTab('pipeline')}>Pipeline</button>

// App.tsx — renderização atual
{activeTab === 'pipeline' && <Pipeline ... />}
```

[Source: docs/architecture/system-architecture.md#application-architecture]

### Target: React Router v7 com Outlet

```typescript
// main.tsx
import { BrowserRouter } from 'react-router-dom'
root.render(
  <BrowserRouter>
    <AuthProvider>
      <App />
    </AuthProvider>
  </BrowserRouter>
)

// App.tsx — target
import { Routes, Route } from 'react-router-dom'
function App() {
  // fetchData() e estado de dados permanecem aqui
  // activeTab useState REMOVIDO
  return (
    <div className="flex">
      <Sidebar />
      <main>
        <Routes>
          <Route path="/" element={<Dashboard ... />} />
          <Route path="/okrs" element={<Okrs ... />} />
          <Route path="/pipeline" element={<Pipeline ... />} />
          {/* ... 6 rotas restantes */}
        </Routes>
      </main>
    </div>
  )
}

// Sidebar.tsx — target
import { useNavigate, useLocation } from 'react-router-dom'
const navigate = useNavigate()
const location = useLocation()
const isActive = (path: string) => location.pathname === path
<button onClick={() => navigate('/pipeline')} className={isActive('/pipeline') ? 'bg-blue-600' : ''}>
  Pipeline
</button>
```

### Animações com React Router

```typescript
// Para preservar page transitions com framer-motion:
import { AnimatePresence } from 'framer-motion'
import { useLocation } from 'react-router-dom'

const location = useLocation()
<AnimatePresence mode="wait">
  <Routes location={location} key={location.pathname}>
    {/* routes */}
  </Routes>
</AnimatePresence>
```

### IMPORTANTE: Não refatorar o estado de dados

O App.tsx tem `fetchData()` com 7 queries paralelas e os dados passados por prop drilling. Esta estrutura será refatorada em Stories 1.2.2 e 1.2.3 (Zustand + Services). Nesta story, só migrar o `activeTab` para routing — não tocar no estado de dados.
