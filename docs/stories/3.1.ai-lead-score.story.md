# Story 3.1 — AI Lead Score no Pipeline

**Status:** Ready for Review | **Sprint CEO-OS-1** | **Agente:** @dev | **Esforço:** 3h

---

## Story Statement

**Como** CEO utilizando o Pipeline do Nexus OS,
**Quero** ver um score de prioridade gerado por IA em cada card de lead,
**Para que** eu saiba instantaneamente quais leads exigem atenção imediata, sem precisar ler todos os cards manualmente.

---

## Contexto

O Pipeline já exibe 7 colunas Kanban com leads filtrados por business unit. Os cards mostram nome, produto, tempo no pipeline e valor. O CEO precisa de mais de 1–2 minutos para identificar qual lead priorizar — especialmente quando há 10+ leads ativos.

A integração com Gemini já existe em `pages/AIAdvisor.tsx` usando `GoogleGenAI` da biblioteca `@google/genai`. O modelo padrão é `AI_MODELS.FLASH` (`gemini-2.5-flash`) definido em `constants.ts`. O `VITE_GEMINI_API_KEY` está configurado no ambiente.

## Acceptance Criteria

- [ ] **AC-1:** Cada card do Pipeline tem um botão "AI Score" (ícone Sparkles) que ao clicar chama o Gemini
- [ ] **AC-2:** O Gemini analisa: `status`, `value`, `days in pipeline`, `painPoint`, `nextAction`, `product`, `businessUnit` e retorna um score de 0–100 + uma sugestão de próxima ação (máx. 1 frase)
- [ ] **AC-3:** O score é exibido como badge colorido no card: ≥70 = verde (emerald), 40–69 = amarelo (amber), <40 = vermelho (rose)
- [ ] **AC-4:** A sugestão de próxima ação é exibida abaixo do badge (texto pequeno, itálico)
- [ ] **AC-5:** Score é cacheado em memória (Zustand) por `lead.id` — chamada Gemini não é repetida para o mesmo lead na mesma sessão
- [ ] **AC-6:** Durante o carregamento, o botão exibe um spinner (Loader2); em caso de erro da API, exibe toast de erro e não quebra o card
- [ ] **AC-7:** A chamada Gemini ocorre **somente sob demanda** (clique do usuário) — nunca no render automático do card
- [ ] **AC-8:** A `VITE_GEMINI_API_KEY` nunca é exposta em logs ou console.log
- [ ] **AC-9:** Zero regressão no Pipeline: os cards existentes (cor, badges, navegação de status) continuam funcionando

## Dev Notes

### Padrão de Integração Gemini (extraído de AIAdvisor.tsx)

```typescript
// Importação — mesma biblioteca já usada no projeto
import { GoogleGenAI } from "@google/genai";
import { AI_MODELS } from '../constants'; // AI_MODELS.FLASH = 'gemini-2.5-flash'

// Inicialização
const ai = new GoogleGenAI({ apiKey: import.meta.env.VITE_GEMINI_API_KEY });

// Chamada (padrão do AIAdvisor.tsx)
const response = await ai.models.generateContent({
  model: AI_MODELS.FLASH,
  contents: prompt,
});
const text = response.text();
```

```typescript
const prompt = `

{"score": <número de 0 a 100 indicando urgência/prioridade>, "suggestion": "<ação recomendada em 1 frase curta>"}
`;
```

### Constraint de performance

A chamada Gemini **não pode** ser feita no render do card. Apenas no `onClick` do botão. Isso evita N chamadas simultâneas ao carregar o Pipeline com 20+ leads.

[Source: EPIC-3-ceo-os-intelligence.md#risk-mitigation]

### Sem mudança no schema do banco

Esta story não requer nenhuma migration SQL. Os dados do lead são apenas lidos da tabela `leads` existente — o score vive apenas em memória (Zustand).
