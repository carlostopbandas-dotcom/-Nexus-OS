# Story 1.0.2 — Mover credenciais Supabase para `.env.local`

**Status:** InReview | **Sprint 0** | **Agente:** @dev | **Esforço:** 20min

> **Segurança crítica:** A Supabase anon key está hardcoded em `lib/supabase.ts`. Qualquer pessoa com acesso ao código ou ao bundle compilado pode acessar o banco de produção. Este é o vetor de ataque P0 ativo identificado no Brownfield Discovery.

---

## Story Statement

**Como** Carlos (CEO / único usuário do Nexus OS),
**Quero** que as credenciais do Supabase sejam movidas para variáveis de ambiente (`.env.local`),
**Para que** as credenciais não fiquem expostas no código fonte e não sejam acidentalmente commitadas no repositório git.

---

## Contexto

**Estado atual — Vetor de ataque ativo:**
```typescript
// lib/supabase.ts:4-5 — EXPOSTO HOJE
const supabaseUrl = 'https://zfmfevpdcwcwatpkttnr.supabase.co'
const supabaseAnonKey = 'sb_publishable_BD0fsd916WaTIoCnLjT6pQ_tQozy6tz'
```

Qualquer pessoa que inspecione o código fonte ou os DevTools do browser pode extrair estas credenciais e fazer queries diretas ao banco de produção (sem RLS ativo, isso significa acesso total a todos os dados).

Débitos remediados: **TD-C1** (credenciais hardcoded) e parcialmente **TD-C2** (GEMINI_API_KEY também está exposta em `vite.config.ts:15-16`).

[Source: docs/architecture/technical-debt-assessment.md#td-c1-td-c2]

### Padrão Vite para env vars

```typescript
// lib/supabase.ts — estado target
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY

if (!supabaseUrl || !supabaseAnonKey) {
  throw new Error('VITE_SUPABASE_URL e VITE_SUPABASE_ANON_KEY são obrigatórias')
}

export const supabase = createClient(supabaseUrl, supabaseAnonKey)
```

### vite.config.ts — remover define

```typescript
// vite.config.ts — REMOVER este bloco:
define: {
  GEMINI_API_KEY: JSON.stringify(process.env.GEMINI_API_KEY || ''),
}
// Variáveis VITE_* são automaticamente expostas pelo Vite via import.meta.env
```

[Source: docs/architecture/system-architecture.md#tech-stack]

### Verificação pós-mudança

Após a mudança, verificar no browser:
1. DevTools → Network → requests ao Supabase continuam funcionando
2. DevTools → Sources → a anon key **não aparece** no bundle (ainda aparece como env var VITE_*, isso é esperado para client-side)

> **Nota:** A anon key Supabase (`sb_publishable_*`) é um token "anon" que é design para ser semi-público — a segurança real virá com RLS (Story 1.1.3). O objetivo desta story é remover do código fonte e evitar commit acidental.
