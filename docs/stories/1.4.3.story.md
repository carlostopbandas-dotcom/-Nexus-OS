# Story 1.4.3 — Consolidar `StoreMetric` em `types.ts`

**Status:** Ready | **Sprint 4 — Quality & DevOps** 🟡 | **Agente:** @dev | **Esforço:** 30min

---

## Story Statement

**Como** desenvolvedor do Nexus OS,
**Quero** ter uma única definição de `StoreMetric` em `types.ts`,
**Para que** não haja interfaces duplicadas e conflitantes entre `App.tsx` e `Dashboard.tsx`.

---

## Contexto

A interface `StoreMetric` foi definida em dois lugares: `App.tsx` e `Dashboard.tsx`. Com o tipo `number | string` para campos financeiros (sales, spend, roas), há risco de cálculos silenciosamente incorretos quando o banco retorna string e o código trata como number.

[Source: docs/architecture/db-specialist-review.md#agr-2]
[Source: docs/architecture/frontend-spec.md — UX-HIGH-2]

---

## Acceptance Criteria

- [ ] **AC-1:** `StoreMetric` definida **apenas** em `types.ts` (ou `src/types/index.ts`)
- [ ] **AC-2:** `sales`, `spend`, `roas` tipados como `number` (não `number | string`)
- [ ] **AC-3:** Todas as referências a `StoreMetric` importam de `types.ts`
- [ ] **AC-4:** Definições duplicadas removidas de `App.tsx` e `Dashboard.tsx`
- [ ] **AC-5:** `npm run typecheck` passa após as mudanças

---

## Tasks / Subtasks

- [ ] **Task 1 — Localizar todas as definições de StoreMetric** (AC: 1)
  - `grep -r "StoreMetric" src/ components/ pages/`

- [ ] **Task 2 — Consolidar em `types.ts`** (AC: 1, 2)
  ```typescript
  // types.ts
  export interface StoreMetric {
    id: string
    store_name: string
    sales: number      // ← era number | string
    spend: number      // ← era number | string
    roas: number       // ← era number | string
    date: string
    created_at: string
    updated_at?: string
  }
  ```

- [ ] **Task 3 — Remover duplicatas** (AC: 3, 4)
  - Remover de `App.tsx` e `Dashboard.tsx`
  - Adicionar imports de `types.ts`

- [ ] **Task 4 — Verificar typecheck** (AC: 5)
  - Possível que mudar `number | string` para `number` cause erros onde o código tentava usar como string
  - Corrigir: adicionar parsing/conversão onde necessário

- [ ] **Task 5 — Commit**
  - `git commit -m "refactor: consolidar StoreMetric em types.ts [Story 1.4.3]"`
