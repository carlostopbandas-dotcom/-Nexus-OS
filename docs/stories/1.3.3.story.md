# Story 1.3.3 — Criar `design-tokens.css` (Single Source of Truth)

**Status:** Ready for Review | **Sprint 3 — Frontend System** 🟡 | **Agente:** @ux-design-expert | **Esforço:** 2h

---

## Story Statement

**Como** desenvolvedor do Nexus OS,
**Quero** ter um arquivo CSS com todas as variáveis de design (cores, tipografia, espaçamento, z-index, sombras),
**Para que** qualquer mudança visual seja feita em um único lugar e propague consistentemente para todo o app.

---

## Contexto

O Nexus OS tem ~50 valores de cor únicos, 8+ font sizes, 6 border-radius values, e z-index manuais espalhados por todo o codebase. Sem tokens centralizados:
- Qualquer mudança visual requer busca em 200+ linhas
- Redesign é praticamente impossível
- Inconsistências visuais são inevitáveis

Débito remediado: **NEW-UX-1** (zero design tokens — arquitetura visual frágil).

### tailwind.config.js — extend com tokens

```javascript
// tailwind.config.js
theme: {
  extend: {
    colors: {
      brand: {
        200: 'var(--color-brand-200)',
        500: 'var(--color-brand-500)',
        700: 'var(--color-brand-700)',
      },
      surface: 'var(--color-surface)',
      sidebar: 'var(--color-sidebar)',
    },
    borderRadius: {
      xs: 'var(--radius-xs)',
      sm: 'var(--radius-sm)',
      md: 'var(--radius-md)',
      lg: 'var(--radius-lg)',
      xl: 'var(--radius-xl)',
    },
    zIndex: {
      dropdown: 'var(--z-dropdown)',
      sidebar:  'var(--z-sidebar)',
      voice:    'var(--z-voice)',
      modal:    'var(--z-modal)',
      toast:    'var(--z-toast)',

    boxShadow: {
      sm: 'var(--shadow-sm)',
      md: 'var(--shadow-md)',
      lg: 'var(--shadow-lg)',
      xl: 'var(--shadow-xl)',
    },
  },
}
```

### Importante: não migrar o código existente nesta story

Esta story APENAS cria os tokens e os conecta ao Tailwind. A substituição das classes `text-[9px]` por `text-nano`, `z-[110]` por `z-modal`, etc. deve acontecer incrementalmente nas stories de componentes (1.3.5, 1.3.6) e de acessibilidade (1.3.8).
