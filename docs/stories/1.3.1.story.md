# Story 1.3.1 — Screenshot baseline das 9 páginas

**Status:** Ready | **Sprint 3 — Frontend System** 🟡 | **Agente:** @dev | **Esforço:** 30min

> **Pré-requisito obrigatório do Sprint 3:** Esta story DEVE ser executada ANTES de qualquer mudança de Tailwind/CSS. O baseline visual é a referência de "idêntico ou melhor" que deve ser mantido ao final de cada sprint.

---

## Story Statement

**Como** desenvolvedor trabalhando na migração de Tailwind CDN → npm,
**Quero** ter screenshots do visual atual das 9 páginas do app antes de qualquer mudança de CSS,
**Para que** seja possível comparar visualmente antes/depois e garantir zero regressão visual.

---

## Contexto

O Princípio de Execução do EPIC-1 é: **"A interface deve estar visualmente idêntica (ou melhor) ao final de cada sprint."**

Antes de migrar Tailwind CDN → npm (Story 1.3.2), precisamos de uma baseline visual documentada. Esta é a referência objetiva para Story 1.3.9 (validação visual).

[Source: docs/stories/EPIC-1-technical-remediation.md#principio-de-execucao]
[Source: docs/architecture/frontend-spec.md#4-inventario-de-paginas]
