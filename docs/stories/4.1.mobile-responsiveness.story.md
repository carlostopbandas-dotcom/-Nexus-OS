# Story 4.1 — Mobile Responsiveness: Sidebar Drawer

**Status:** InProgress | **Sprint CEO-OS-2** | **Agente:** @dev | **Esforço:** 1h

---

## Story Statement

**Como** CEO usando o Nexus OS no celular,
**Quero** que o app seja responsivo com sidebar escondida por padrão e acessível via menu hamburger,
**Para que** eu possa navegar normalmente no mobile sem o conteúdo ficar escondido pela sidebar fixa.

---

## Acceptance Criteria

- [ ] **AC-1:** Sidebar oculta no mobile (< md) por padrão — conteúdo ocupa tela inteira
- [ ] **AC-2:** Header mobile fixo no topo com botão hamburger (Menu icon) + logo "NEXUS OS"
- [ ] **AC-3:** Toque no hamburger abre sidebar como drawer (slide-in da esquerda)
- [ ] **AC-4:** Backdrop escuro cobre conteúdo quando drawer aberto — toque fecha
- [ ] **AC-5:** Clicar em qualquer item de navegação fecha o drawer no mobile
- [ ] **AC-6:** Desktop (>= md) sem alteração — sidebar sempre visível, sem mobile header

---

## Tasks

- [ ] **Task 1 — App.tsx:** Adicionar `useState<boolean>` sidebarOpen + passar props para Sidebar + trocar `ml-72` por `ml-0 md:ml-72` + mobile header
- [ ] **Task 2 — Sidebar.tsx:** Aceitar `isOpen/onClose` props + `transition-transform` + backdrop overlay + fechar ao navegar
