# Story 1.3.4 — Instalar shadcn/ui

**Status:** Ready | **Sprint 3 — Frontend System** 🟡 | **Agente:** @dev | **Esforço:** 1h

> **Pré-requisito:** Tailwind npm (1.3.2) e design-tokens.css (1.3.3) devem estar prontos. shadcn/ui não funciona com Tailwind CDN.

---

## Story Statement

**Como** desenvolvedor do Nexus OS,
**Quero** ter o shadcn/ui configurado no projeto,
**Para que** componentes com acessibilidade (Radix UI) e focus trap nativos possam ser usados nas Stories 1.3.5 e 1.3.6.

---

## Contexto

shadcn/ui provê componentes baseados em Radix UI primitives com:
- Acessibilidade nativa (WCAG AA): focus trap, aria-labels, ESC handlers
- Componentes copiados para o projeto (não biblioteca externa)
- Integração nativa com Tailwind CSS

ADR validado: **ADR-3** (shadcn/ui aprovado com ressalva: manter framer-motion para animações).

[Source: docs/architecture/ux-specialist-review.md#adrs — ADR-3 APROVADO com ressalva]

## Dev Notes

### shadcn init prompts esperados

```
✔ Which style would you like to use? › New York
✔ Which color would you like to use as the base color? › Slate
✔ Would you like to use CSS variables? › yes
```

"New York" tem visual mais polido e compacto — melhor fit para o Nexus OS premium.

### Ressalva do ADR-3: framer-motion permanece

[Source: docs/architecture/ux-specialist-review.md#adrs]
shadcn usa `tailwindcss-animate` para suas animações internas. **Não substituir** as animações framer-motion existentes nas page transitions e card hovers. As duas bibliotecas coexistem:
- framer-motion → page transitions, card hover lift, VoiceAssistant
- tailwindcss-animate (shadcn) → Dialog open/close, Toast entrance

### O que shadcn instala (apenas estrutura)

shadcn não instala componentes automaticamente — apenas configura o projeto. Componentes são adicionados manualmente via `npx shadcn@latest add dialog`, etc. Isso acontece nas Stories 1.3.5 e 1.3.6.
