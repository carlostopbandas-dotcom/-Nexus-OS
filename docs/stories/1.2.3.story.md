# Story 1.2.3 — Criar service layer (7 services)

**Status:** Done | **Sprint 2 — Architecture & DB Foundation** 🟢 | **Agente:** @dev | **Esforço:** 6h

---

## Story Statement

**Como** desenvolvedor do Nexus OS,
**Quero** que as queries ao Supabase sejam encapsuladas em services dedicados,
**Para que** nenhuma página ou componente faça queries diretas ao banco — apenas via service layer — facilitando testes, manutenção e futuros mocks.

---

## Contexto

Hoje as queries Supabase estão espalhadas pelo App.tsx, pages/*.tsx e componentes. Não há separação entre lógica de dados e lógica de UI. Isso torna impossível:
- Escrever testes unitários sem banco real
- Reusar queries entre componentes
- Mudar o provider de dados sem tocar nos componentes

Débito remediado: **TD-H2** (queries diretas Supabase nos componentes, sem service layer).

### Padrão de service (exemplo: leadsService)

```typescript
// services/leadsService.ts
import { supabase } from '@/lib/supabase'
import type { Lead } from '@/types'

type ServiceResult<T> = { data: T | null; error: string | null }

export const leadsService = {
  async getAll(): Promise<ServiceResult<Lead[]>> {
    const { data, error } = await supabase
      .from('leads')
      .select('*')
      .order('created_at', { ascending: false })
    return { data: data ?? null, error: error?.message ?? null }
  },

  async getById(id: string): Promise<ServiceResult<Lead>> {
    const { data, error } = await supabase
      .from('leads')
      .select('*')
      .eq('id', id)
      .single()
    return { data: data ?? null, error: error?.message ?? null }
  },

  async create(lead: Omit<Lead, 'id' | 'created_at'>): Promise<ServiceResult<Lead>> {
    const { data, error } = await supabase
      .from('leads')
      .insert(lead)
      .select()
      .single()
    return { data: data ?? null, error: error?.message ?? null }
  },

  async update(id: string, updates: Partial<Lead>): Promise<ServiceResult<Lead>> {
    const { data, error } = await supabase
      .from('leads')
      .update(updates)
      .eq('id', id)
      .select()
      .single()
    return { data: data ?? null, error: error?.message ?? null }
  },

  async delete(id: string): Promise<ServiceResult<null>> {
    const { error } = await supabase.from('leads').delete().eq('id', id)
    return { data: null, error: error?.message ?? null }
  },
}
```
