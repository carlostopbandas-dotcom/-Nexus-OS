# Story 3.2 — AI Pipeline Summary no Dashboard

**Status:** Ready for Review | **Sprint CEO-OS-1** | **Agente:** @dev | **Esforço:** 2h

---

## Story Statement

**Como** CEO visualizando o Dashboard do Nexus OS,
**Quero** ver um resumo inteligente do meu pipeline gerado por IA,
**Para que** eu tenha em segundos as 3 maiores oportunidades, os leads em risco e um foco de ação — sem precisar analisar cada card manualmente.

---

## Contexto

O Dashboard já exibe KPIs globais (faturamento, ROAS, pipeline, conversão) e o YTD acumulado. Falta uma camada de inteligência que interprete o estado do pipeline e entregue insights acionáveis diretamente na tela principal.

O padrão de integração Gemini foi estabelecido na Story 3.1 via `hooks/useLeadAIScore.ts`:
- `GoogleGenAI` de `@google/genai`
- Modelo `AI_MODELS.FLASH` (`gemini-2.5-flash`) de `constants.ts`
- `response.text` é getter (não método)
- API key via `import.meta.env.VITE_GEMINI_API_KEY`
- Cache de sessão em Zustand
- Chamada apenas on-demand (nunca no render)

O widget é inserido entre o bloco YTD Acumulado (linha ~340) e os Painéis condicionais (linha ~342) — visível em todos os tabs (Overview, 3D Digital, Grupo VcChic).

## Dev Notes

### Padrão de Integração Gemini (estabelecido em Story 3.1)

```typescript
// MESMO PADRÃO de hooks/useLeadAIScore.ts
import { GoogleGenAI } from '@google/genai';
import { AI_MODELS } from '../constants'; // AI_MODELS.FLASH = 'gemini-2.5-flash'

const ai = new GoogleGenAI({ apiKey: import.meta.env.VITE_GEMINI_API_KEY });
const response = await ai.models.generateContent({
  model: AI_MODELS.FLASH,
  contents: prompt,
});
const text = response.text ?? ''; // response.text é GETTER, não método
const parsed = JSON.parse(text.trim());
```

[Source: hooks/useLeadAIScore.ts — padrão já testado e aprovado]

### Prompt recomendado para o summary

```typescript
const byStatus = leads.reduce((acc, l) => {
  acc[l.status] = (acc[l.status] ?? 0) + 1;
  return acc;
}, {} as Record<string, number>);

const top5 = [...leads]
  .sort((a, b) => b.value - a.value)
  .slice(0, 5)
  .map(l => `${l.name} (${l.status}, R$${l.value}, ${l.businessUnit ?? ''})`);

const semAcao = leads.filter(l => !l.nextAction).map(l => l.name);

const prompt = `Analise o pipeline de vendas abaixo e retorne SOMENTE um JSON válido, sem markdown, sem explicação:

Pipeline:
- Total de leads: ${leads.length}
- Por status: ${JSON.stringify(byStatus)}
- Top 5 por valor: ${top5.join(' | ')}
- Leads sem próxima ação: ${semAcao.length > 0 ? semAcao.join(', ') : 'nenhum'}

Retorne:
{
  "opportunities": ["<oportunidade 1>", "<oportunidade 2>", "<oportunidade 3>"],
  "risks": ["<risco 1>", "<risco 2>"],
  "focus": "<uma recomendação de foco para o CEO em 1 frase>"
}`;
```

### Constraint de performance

Mesma regra da Story 3.1: **nunca chamar Gemini no render**. O snapshot do pipeline pode ter muitos leads — o prompt é construído de forma condensada (apenas totais + top 5) para não exceder tokens.

[Source: EPIC-3-ceo-os-intelligence.md#risk-mitigation]

### Sem mudança no schema do banco

Esta story não requer migration SQL. Todos os dados vêm do Zustand `leads[]` já carregado.
