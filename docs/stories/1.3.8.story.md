# Story 1.3.8 — Adicionar aria-labels WCAG Level A em todos os inputs

**Status:** Ready for Review | **Sprint 3 — Frontend System** 🟡 | **Agente:** @dev | **Esforço:** 2h

---

## Story Statement

**Como** Carlos (CEO / único usuário do Nexus OS),
**Quero** que todos os inputs, botões de ação e a sidebar tenham aria-labels corretos,
**Para que** o app passe nos critérios WCAG Level A e seja utilizável com leitores de tela.

---

## Contexto

O Nexus OS tem zero compliance WCAG. As falhas Level A identificadas são bloqueantes para qualquer usuário com necessidade especial, e são boas práticas independente do público-alvo.

Débito remediado: **AGR-UX-2** (acessibilidade 1.5/10), **UX-CRIT-3** (sem acessibilidade).

[Source: docs/architecture/ux-specialist-review.md#agr-ux-2]
[Source: docs/architecture/ux-specialist-review.md#5-acessibilidade]

---

## Acceptance Criteria

- [ ] **AC-1:** Todos os `<input>` têm `aria-label` ou `<label htmlFor>` associado
- [ ] **AC-2:** Todos os `<button>` com apenas ícone têm `aria-label` descritivo
- [ ] **AC-3:** `<Sidebar>` tem `<nav aria-label="Navegação principal">`
- [ ] **AC-4:** `<main>` tem `role="main"` ou landmark semântico
- [ ] **AC-5:** Focus visible CSS adicionado: `*:focus-visible { outline: 2px solid ... }`
- [ ] **AC-6:** Inputs migrados para o `<Input>` atom da Story 1.3.5 (com aria-label obrigatório no tipo)
- [ ] **AC-7:** WCAG Level A criteria 1.3.1, 2.1.1, 4.1.2 atendidos

## Dev Notes

### WCAG failures confirmados

```
1.3.1 Info and Relationships (Level A) → inputs sem label
2.1.1 Keyboard (Level A) → modais sem trapfocus (corrigido em 1.3.6)
2.4.3 Focus Order (Level A) → z-index manual quebra focus order
4.1.2 Name, Role, Value (Level A) → botões sem aria-label
```

[Source: docs/architecture/ux-specialist-review.md#agr-ux-2]

### Exemplos de fix

```tsx
// ❌ ANTES:
<input placeholder="Nome do lead" onChange={...} />
<button><Trash2 /></button>

// ✅ DEPOIS:
<Input aria-label="Nome do lead" placeholder="Nome do lead" onChange={...} />
<button aria-label="Excluir lead"><Trash2 /></button>

// Sidebar:
// ❌ <div className="..."> menu items </div>
// ✅ <nav aria-label="Navegação principal"> menu items </nav>
```

[Source: docs/architecture/ux-specialist-review.md#5-acessibilidade]

### Contraste de texto (Level AA — não obrigatório nesta story)

Um ajuste de cor identificado para compliance Level AA:
```css
/* text-slate-400 (#94A3B8) sobre white = 2.5:1 ❌ */
/* Fix: usar text-slate-500 (#64748B) = 4.5:1 ✅ */
```
Este ajuste pode ser feito oportunisticamente mas não é o foco desta story.
