# Story 2.7 — Integração Google Calendar bidirecional

## Story

**Como** CEO,  
**Quero** ver e criar eventos do meu Google Calendar diretamente no Nexus OS,  
**Para** ter minha agenda centralizada sem precisar alternar entre ferramentas.

## Context

A tabela `events` existe desde a Story 0 com campos simples (`title`, `start_time`, `end_time`, `type`, `attendees`, `day_offset`). A página `Routine.tsx` consome `eventsService` para exibir a agenda do CEO.

Esta story adiciona sincronização bidirecional com o Google Calendar:
- **Import** (Google → Nexus): eventos do Google Calendar upserted em `events` usando `google_event_id` como chave de deduplicação
- **Export** (Nexus → Google): eventos criados no Nexus (onde `google_event_id IS NULL`) criados no Google Calendar e atualizados com o `google_event_id` retornado

A sincronização é disparada manualmente pelo CEO via UI (cron automático é fase futura). Segue o padrão exato da integração Facebook Ads (Story 2.2): Supabase Edge Function + `CRON_SECRET` + log em tabela dedicada.

**Conflitos:** Google Calendar sempre vence. Se um evento foi modificado nos dois lados, a versão do Google prevalece na próxima importação.

## Acceptance Criteria

### AC-1: Migration 012 — schema de sincronização
- [ ] Novo arquivo `supabase/migrations/012_google_calendar.sql`
- [ ] Adiciona coluna `google_event_id text` (nullable) à tabela `events`:
  ```sql
  ALTER TABLE events ADD COLUMN IF NOT EXISTS google_event_id text;
  CREATE UNIQUE INDEX IF NOT EXISTS idx_events_google_id
    ON events(google_event_id) WHERE google_event_id IS NOT NULL;
  ```
- [ ] Cria tabela `calendar_sync_log` (mesmo padrão de `ads_sync_log`):
  ```sql
  CREATE TABLE IF NOT EXISTS calendar_sync_log (
    id          uuid DEFAULT gen_random_uuid() PRIMARY KEY,
    status      text NOT NULL CHECK (status IN ('success', 'error', 'partial')),
    imported    integer DEFAULT 0,
    exported    integer DEFAULT 0,
    error_msg   text,
    synced_at   timestamptz DEFAULT now()
  );
  ```
- [ ] RLS habilitado em `calendar_sync_log`: CEO ALL, nenhuma outra role
- [ ] Script de rollback incluído no arquivo (comentário `-- ROLLBACK`)

### AC-2: Supabase Edge Function `google-calendar-sync`
- [ ] Novo arquivo `supabase/functions/google-calendar-sync/index.ts`
- [ ] Segredos lidos do ambiente: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REFRESH_TOKEN_CEO`, `CRON_SECRET`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`
- [ ] Autenticação: `x-cron-secret` header ou Bearer token de sessão CEO válida
- [ ] Se segredos Google não configurados: retorna 503 com mensagem explicativa
- [ ] **Fluxo OAuth2:** troca `refresh_token` por `access_token` via `https://oauth2.googleapis.com/token`
- [ ] **Import (Google → Nexus):**
  - Busca eventos dos próximos 90 dias via `GET https://www.googleapis.com/calendar/v3/calendars/primary/events`
  - Faz `upsert` em `events` com `onConflict: 'google_event_id'` — atualiza `title`, `start_time`, `end_time`
  - Conta eventos importados/atualizados
- [ ] **Export (Nexus → Google):**
  - Lê `events` WHERE `google_event_id IS NULL` e `start_time > now()`
  - Para cada evento: `POST https://www.googleapis.com/calendar/v3/calendars/primary/events`
  - Após sucesso: `UPDATE events SET google_event_id = '...' WHERE id = '...'`
  - Conta eventos exportados
- [ ] Grava resultado em `calendar_sync_log`
- [ ] Retorna JSON `{ imported: N, exported: N, synced_at: '...' }`

### Troca OAuth2 (refresh_token → access_token)
```typescript
const tokenRes = await fetch('https://oauth2.googleapis.com/token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
  body: new URLSearchParams({
    client_id: GOOGLE_CLIENT_ID!,
    client_secret: GOOGLE_CLIENT_SECRET!,
    refresh_token: GOOGLE_REFRESH_TOKEN_CEO!,
    grant_type: 'refresh_token',
  }),
})
const { access_token } = await tokenRes.json()
```

### Listagem de eventos do Google Calendar
```typescript
const calRes = await fetch(
  `https://www.googleapis.com/calendar/v3/calendars/primary/events?timeMin=${now}&maxResults=100&singleEvents=true&orderBy=startTime`,
  { headers: { Authorization: `Bearer ${access_token}` } }
)
const { items } = await calRes.json()
```

### Upsert com deduplicação por `google_event_id`
```typescript
await supabase.from('events').upsert(
  items.map(e => ({
    title: e.summary ?? '(sem título)',
    start_time: e.start.dateTime ?? e.start.date,
    end_time: e.end?.dateTime ?? e.end?.date,
    google_event_id: e.id,
  })),
  { onConflict: 'google_event_id' }
)
```
