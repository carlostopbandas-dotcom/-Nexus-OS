# Story 2.9 — Onboarding de usuário

## Story

**Como** usuário que acabou de receber acesso ao Nexus OS,  
**Quero** ver uma mensagem de boas-vindas explicando o sistema no meu primeiro login,  
**Para** entender rapidamente como navegar e o que está disponível para mim.

## Context

A tabela `user_profiles` já possui a coluna `onboarded_at timestamptz` (migration 006). O banner é exibido no Dashboard no primeiro acesso e descartado ao clicar em "Entendido" — o descarte grava `onboarded_at = now()`. A mensagem varia por role.

## Acceptance Criteria

### AC-1: `onboardingService.ts`
- [ ] Novo arquivo `services/onboardingService.ts`
- [ ] `isOnboarded()`: lê `user_profiles.onboarded_at` do usuário atual — retorna `boolean`
- [ ] `markOnboarded()`: faz `UPDATE user_profiles SET onboarded_at = now()` para o usuário atual
- [ ] Pattern `ServiceResult<T>` consistente

### AC-2: `OnboardingBanner.tsx`
- [ ] Novo arquivo `components/auth/OnboardingBanner.tsx`
- [ ] Banner exibido apenas quando `isOnboarded() === false`
- [ ] Mensagem contextual por role (`ceo`, `gestor_vcchic`, `vendedor_sdr`, `assistente`)
- [ ] Botão "Entendido" → chama `markOnboarded()` → desmonta o banner
- [ ] Design: `bg-blue-50 border border-blue-100 rounded-2xl p-4`

### AC-3: Integração em `Dashboard.tsx`
- [ ] `OnboardingBanner` adicionado no topo de `Dashboard.tsx`

### AC-4: Testes `onboardingService.test.ts`
- [ ] `isOnboarded` — retorna `true` quando `onboarded_at` preenchido
- [ ] `isOnboarded` — retorna `false` quando `onboarded_at` null
- [ ] `markOnboarded` — atualiza com sucesso
