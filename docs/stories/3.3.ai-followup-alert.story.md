# Story 3.3 — AI Follow-up Alert em Clientes Ativos

**Status:** Done | **Sprint CEO-OS-1** | **Agente:** @dev | **Esforço:** 2h

---

## Story Statement

**Como** CEO acompanhando os Clientes Ativos no Nexus OS,
**Quero** ver um alerta automático nos cards de clientes sem follow-up recente, com uma sugestão de mensagem WhatsApp gerada por IA,
**Para que** eu nunca perca um cliente por falta de acompanhamento — identificando rapidamente quem precisa de atenção e com o que falar.

---

## Contexto

A página `/clients` já exibe cards com nome, produto, módulo, valor, status de pagamento e data de follow-up. O CEO consegue ver quando o follow-up está registrado — mas não recebe nenhum sinal quando está atrasado ou ausente.

O padrão de integração Gemini foi consolidado em Stories 3.1 e 3.2:
- `GoogleGenAI` de `@google/genai`
- Modelo `AI_MODELS.FLASH` (`gemini-2.5-flash`) de `constants.ts`
- `response.text` é getter (não método)
- API key via `import.meta.env.VITE_GEMINI_API_KEY`
- Cache de sessão em Zustand
- Chamada apenas on-demand (clique) — nunca no render

O componente `ClientCard` (dentro de `pages/Clients.tsx`) renderiza cada cliente. O alerta de follow-up deve ser adicionado como sub-componente `AIFollowUpAlert` dentro do `ClientCard`, após o bloco de valor e follow-up (footer do card).

## Acceptance Criteria

- [ ] **AC-1:** Badge de alerta "Sem follow-up" visível automaticamente no `ClientCard` quando `followUpDate` for nulo **ou** anterior a 7 dias — sem chamada à API, sem ação do usuário
- [ ] **AC-2:** Botão "Sugestão IA" (ícone `Sparkles`) aparece no footer do card quando o alerta estiver ativo — ao clicar chama o Gemini com dados do cliente (nome, produto, módulo, dias sem follow-up, paymentStatus)
- [ ] **AC-3:** Gemini retorna JSON `{ urgency: 'alta'|'media'|'baixa', suggestion: string }` — sugestão exibida como texto de mensagem WhatsApp abaixo do badge, com botão "Copiar" (ícone `Copy`)
- [ ] **AC-4:** Badge de urgência colorido conforme retorno do Gemini: `alta` = rose, `media` = amber, `baixa` = slate
- [ ] **AC-5:** Sugestão cacheada em Zustand por `client.id` — chamada Gemini não repetida para o mesmo cliente na mesma sessão
- [ ] **AC-6:** Durante carregamento exibe spinner `Loader2`; erro da API → toast de erro sem quebrar o card; último resultado preservado se existir
- [ ] **AC-7:** Chamada Gemini ocorre **somente sob demanda** (clique) — nunca no render automático do card
- [ ] **AC-8:** `VITE_GEMINI_API_KEY` nunca exposta em logs ou `console.log`
- [ ] **AC-9:** Zero regressão em Clients.tsx: payment status, module progress, botão WhatsApp e busca continuam funcionando

## Dev Notes

### Padrão de Integração Gemini (consolidado nas Stories 3.1 e 3.2)

```typescript
import { GoogleGenAI } from '@google/genai';
import { AI_MODELS } from '../constants'; // AI_MODELS.FLASH = 'gemini-2.5-flash'

const ai = new GoogleGenAI({ apiKey: import.meta.env.VITE_GEMINI_API_KEY });
const response = await ai.models.generateContent({
  model: AI_MODELS.FLASH,
  contents: prompt,
});
const text = response.text ?? ''; // GETTER, não método
const parsed = JSON.parse(text.trim());
```

[Source: hooks/useLeadAIScore.ts + hooks/usePipelineSummary.ts — padrão testado]

### Lógica de detecção de alerta (sem IA)

```typescript
const FOLLOWUP_THRESHOLD_DAYS = 7;

const daysWithoutFollowUp = client.followUpDate
  ? Math.floor((Date.now() - new Date(client.followUpDate + 'T00:00:00').getTime()) / 86_400_000)
  : null;

const needsAlert =
  daysWithoutFollowUp === null || daysWithoutFollowUp >= FOLLOWUP_THRESHOLD_DAYS;
```

### Prompt recomendado para sugestão WhatsApp

```typescript
const daysLabel = daysWithoutFollowUp !== null
  ? `${daysWithoutFollowUp} dias`
  : 'nunca registrado';

const prompt = `Você é um assistente de vendas consultivo. Analise este cliente e gere uma mensagem de follow-up para WhatsApp.

Cliente:
- Nome: ${client.name}
- Produto: ${client.product}
- Módulo atual: ${client.module ?? 'não informado'}
- Dias sem follow-up: ${daysLabel}
- Status de pagamento: ${client.paymentStatus ?? 'Em dia'}

Retorne SOMENTE um JSON válido, sem markdown, sem explicação:
{"urgency": "<alta|media|baixa>", "suggestion": "<mensagem WhatsApp em 2-3 frases, tom consultivo e empático, primeira pessoa>"}`;
```

### Design do componente

```tsx
// Badge de alerta automático (sem IA)
<div className="mt-3 pt-3 border-t border-slate-50">
  <div className="flex items-center justify-between">
    <span className="text-[10px] font-black text-amber-600 uppercase tracking-widest flex items-center gap-1">
      <AlertCircle size={11} />
      {daysWithoutFollowUp !== null ? `Sem follow-up há ${daysWithoutFollowUp}d` : 'Sem follow-up registrado'}
    </span>
    <button onClick={analyze} className="flex items-center gap-1 text-[10px] font-black text-indigo-500 hover:text-indigo-700">
      <Sparkles size={11} /> Sugestão IA
    </button>
  </div>

  {/* Com suggestion */}
  {alert && (
    <div className={`mt-2 p-3 rounded-xl text-[11px] text-slate-600 leading-relaxed ${URGENCY_BG[alert.urgency]}`}>
      <p>{alert.suggestion}</p>
      <button onClick={copyToClipboard} className="mt-2 flex items-center gap-1 text-[10px] font-bold text-slate-500 hover:text-slate-800">
        <Copy size={11} /> Copiar mensagem
      </button>
    </div>
  )}
</div>

// Cores de urgência
const URGENCY_BG = { alta: 'bg-rose-50', media: 'bg-amber-50', baixa: 'bg-slate-50' };
const URGENCY_TEXT = { alta: 'text-rose-600', media: 'text-amber-600', baixa: 'text-slate-500' };
```
