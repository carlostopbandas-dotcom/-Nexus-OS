**Status:** Ready | **Sprint 1 — Security Foundation** 🔴 | **Agente:** @dev | **Esforço:** 3h

[Source: docs/architecture/system-architecture.md#ai-layer]

### Estratégia de proxy para WebSocket

O WebSocket bidirecional é complexo para proxying. Há duas abordagens possíveis:

**Opção A — Proxy de token (preferida):**
- Edge Function gera um token de sessão temporário para a API Gemini
- Frontend usa o token para conectar diretamente (não expõe a API key permanente)

**Opção B — Proxy REST para inicialização:**
- Edge Function autentica e retorna endpoint WebSocket temporário
- Frontend conecta ao WebSocket do Gemini com token de curta duração

**Opção C — Proxy completo via Edge Function:**
- Frontend conecta à Edge Function via WebSocket
- Edge Function faz proxy de tudo para o Gemini
- Mais complexo, mais latência

> **Recomendação:** Verificar se o SDK `@google/genai` suporta tokens temporários (API keys de sessão). Se sim, Opção A é a mais simples. Se não, avaliar Opção B vs C baseado na documentação atual do Gemini Live API.

### Estrutura da Edge Function (base)

```typescript
// supabase/functions/gemini-proxy/index.ts
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts'
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'

const GEMINI_API_KEY = Deno.env.get('GEMINI_API_KEY')

serve(async (req) => {
  // CORS
  if (req.method === 'OPTIONS') {
    return new Response('ok', { headers: corsHeaders })
  }

  // Validar autenticação
  const authHeader = req.headers.get('Authorization')
  if (!authHeader) {
    return new Response(JSON.stringify({ error: 'Unauthorized' }), { status: 401 })
  }

  const supabaseClient = createClient(
    Deno.env.get('SUPABASE_URL') ?? '',
    Deno.env.get('SUPABASE_ANON_KEY') ?? '',
    { global: { headers: { Authorization: authHeader } } }
  )

  const { data: { user } } = await supabaseClient.auth.getUser()
  if (!user) {
    return new Response(JSON.stringify({ error: 'Unauthorized' }), { status: 401 })
  }

  // Proxy para Gemini API
  // ... implementação específica baseada no protocolo do VoiceAssistant
})

const corsHeaders = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type',
}
```
